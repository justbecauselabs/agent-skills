# Design Principles

HeroUI Native is built with a few guiding principles that shape component APIs and defaults:

- **Data is central**: Components focus on display and interaction, not business logic.
- **Composition over complexity**: Prefer composable parts over monolithic components.
- **Sensible defaults**: Minimal configuration to get high-quality UI quickly.
- **Consistency**: Shared tokens and patterns keep UI cohesive across screens.
- **Accessible by default**: Keyboard and screen reader support are prioritized.
- **Motion with purpose**: Animations should be subtle and reinforce state changes.
